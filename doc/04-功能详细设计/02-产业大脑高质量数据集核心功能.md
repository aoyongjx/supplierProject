# 自动化网络数据采集与爬取管理系统核心功能

## 功能目标

- 实现智能网联新能源汽车供应链相关互联网数据的自动化、可配置采集，支持自然语言指令发起爬取任务。
- 提供可视化爬取管理入口，实现任务调度、状态监控、异常告警与数据质量校验。
- 采集数据清洗、标准化后，批量写入公司MPP数据库，为产业链分析与供应商画像提供基础数据源。

## 业务流程

1. 用户通过可视化界面或自然语言交互，提交爬取需求，系统自动解析并生成标准化爬取任务。
2. 调度引擎执行爬取任务，通过多源爬虫框架按规则采集行业报告、企业公告、招投标、专利、认证、产能等数据。
3. 对采集数据进行格式标准化、去重、补全、规则校验，标记脏数据并记录异常日志。
4. 清洗后的结构化数据批量写入公司MPP数据库，更新爬取任务状态，并向用户推送任务完成/异常通知。

## 输入输出

- 输入：自然语言爬取指令、可视化配置的爬取任务参数（数据源地址、字段映射、更新频率、过滤规则）
- 输出：结构化标准化供应链数据集（写入MPP数据库）、爬取任务状态报告、数据质量报告、异常告警日志。

## 业务规则

- 爬取任务需遵守目标网站robots协议及相关法律法规，设置合理请求频率。
- 所有采集数据需保留原始来源、采集时间戳、任务ID、来源网址等溯源字段。
- 自然语言爬取指令需经过合规性校验。
- 数据清洗规则可配置，不满足规则的数据标记为“脏数据”，不写入正式库。
- 支持增量爬取与全量更新两种模式，默认按天/周配置增量更新频率。

## 数据关联

- 采集数据为产业链图谱构建、供应商信息库建设提供基础数据源。
- 爬取任务表与MPP数据库业务表通过任务ID建立关联，实现数据溯源。

## 异常处理

- 目标网站访问失败/反爬拦截：自动重试3次，失败则标记异常并推送告警，支持人工重跑。
- 数据格式/字段不匹配：记录脏数据日志，支持人工修正规则后重新清洗入库。
- MPP数据库写入失败：缓存数据并重试，持续失败则触发DBA告警。
- 自然语言指令解析失败：返回错误提示，引导用户补充或修正需求。

## 验收标准

- 支持至少5类典型数据源自动化爬取（企业官网、行业平台、招投标网站、专利/认证平台、新闻资讯）。
- 自然语言指令解析准确率≥90%。
- 爬取任务执行成功率≥99%，异常告警响应时间≤5分钟。
- 数据入库成功率≥99.5%，脏数据率≤0.5%。
- 提供可视化管理界面，支持任务创建、启停、查看、日志查询、重跑。
