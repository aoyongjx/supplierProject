# 使用说明

## 文档定位

本目录是 `asset-inventory-node24` 项目的交付文档中心，用于统一以下内容：

- 业务需求与目标
- 技术架构与数据接口
- 功能详细设计
- 测试用例与验收口径
- 开发参考与工程规范
- 项目进度与风险跟踪

## 推荐阅读顺序

1. `01-AI入口.md`
2. `02-需求和项目介绍/README.md`
3. `03-技术架构设计/README.md`
4. `04-功能详细设计/README.md`
5. `05-测试用例设计/README.md`
6. `06-开发参考/README.md`
7. `07-开发规范/README.md`
8. `08-项目进度/README.md`

## 与代码一致性约束

- 本目录内容应与 `code/asset-inventory-node24` 当前实现保持一致。
- 当页面路由、接口、数据表、鉴权方式发生变化时，需同步更新本目录。
- 需求文档描述“做什么”，架构文档描述“怎么做”，测试文档描述“怎么验收”。

## 维护建议

- 功能变更优先更新 `04-功能详细设计/`。
- 后端路由或数据结构变更优先更新 `03-技术架构设计/05-接口设计.md`、`03-技术架构设计/04-数据库设计.md`。
- 每次版本迭代至少更新一次 `08-项目进度/`。
