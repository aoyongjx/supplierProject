# 部署与运行设计

## 运行模式

1. 开发模式
- 命令：`npm run dev`
- 行为：并行启动前端 Vite 与后端 Express。

2. 稳定演示模式
- 命令：`npm run stable`
- 行为：后端服务 + 前端 preview 固定端口运行，适合演示和联调。

## 端口与地址

- 后端：`3000`
- 前端：`5173`
- OAuth2 回调依赖后端 `3000` 端口（见项目约束）。

## 环境变量

- 数据库：`DB_HOST`、`DB_PORT`、`DB_NAME`、`DB_SCHEMA`、`DB_USER`、`DB_PASSWORD`
- 认证：`AUTH_ENABLED`、`AUTH_BASE_URL` 及相关 client 参数
- 其他：代理、NO_PROXY、模型相关配置

## 初始化与建表

- 服务启动时自动检查并创建核心表结构。
- 首次运行建议先确认数据库连通与权限，再启动服务。

## 运维建议

- 修改配置后先执行健康检查：`GET /api/health`。
- 生产环境建议增加反向代理、日志采集和进程守护。
- 大任务执行前先做小样本预跑，确认模型/外部服务可用。
