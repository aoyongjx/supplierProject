# 鉴权与安全设计

## 鉴权方案

- 采用 OAuth2 授权码模式对接外部 Auth 服务。
- 关键接口：
  - `GET /api/auth/login-url`
  - `GET /api/auth/callback`
  - `GET /api/auth/me`
  - `GET /api/auth/logout-url`
- 业务 API 默认接入 `authMiddleware`。

## Token 与会话处理

- 回调后由后端处理令牌交换并重定向前端。
- 前端通过 token 存储维持登录态，并按需调用 `/api/auth/me`。
- 未登录或 token 失效时跳转登录页。

## 安全策略

1. 输入安全
- 对关键参数进行类型与必填校验（尤其 SQL、任务、模型配置等场景）。

2. 配置安全
- `.env` 管理连接信息与密钥，不入库、不明文暴露到前端。

3. 数据安全
- 高风险操作提供二次确认（前端）与后端校验。

4. 运行安全
- 对外部服务调用设置超时、重试与失败反馈。
