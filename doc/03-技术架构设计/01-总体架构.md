# 总体架构

## 1. 架构风格

当前系统采用“前后端分离 + 单体后端 + 数据库中心 + 外部能力接入”的实用架构：

- 前端：React/Vite 中后台应用，菜单化路由组织多业务域。
- 后端：Express 单服务承载业务 API、任务编排、AI 接口、配置管理。
- 数据：PostgreSQL/SeaboxSQL 作为核心存储。
- 外部：Auth、模型服务、MCP/搜索等外部依赖按需接入。

## 2. 逻辑分层

1. 展示层
- `src/pages/*`
- 负责交互、列表、表单、图谱、会话视图

2. API 编排层
- `server/index.js` 路由入口
- 负责鉴权校验、参数校验、统一响应、错误封装

3. 领域服务层
- 供应商域、供应链域、任务域、知识域、Wiki 域、Agent 域

4. 数据访问层
- `pg` 连接池、事务与 SQL
- 表初始化与索引维护

5. 外部能力层
- OAuth2 Auth
- 模型提供商（OpenAI 兼容）
- LangChain 工具链
- 可选 MCP / 搜索服务

## 3. 数据流说明

- 用户请求从前端发起，经鉴权后进入对应领域 API。
- 领域 API 读写 DB，必要时调用外部服务（模型/网页/MCP）。
- 任务型流程产生状态、日志、质量报告与导入结果。
- AI 流程将 DB/知识库/Wiki 数据组合为上下文，返回回答。

## 4. 技术特征

- 高功能密度：一个应用容纳业务管理、任务执行、知识库、智能体、运维配置。
- 强配置化：菜单可见性、样式、模型、搜索策略都可在系统内管理。
- 可追溯：任务日志、质量报告、同步记录、会话记录可沉淀。
