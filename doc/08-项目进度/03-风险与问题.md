# 风险与问题

## 当前风险

1. 外部依赖风险
- Auth、模型、MCP/搜索等外部服务可用性会直接影响功能体验。

2. 单体服务复杂度风险
- `server/index.js` 承载业务较多，后续可考虑按域拆分。

3. 数据质量风险
- 爬取与导入链路中仍可能出现字段不一致、脏数据、重复数据。

4. 文档同步风险
- 功能迭代较快，若未建立发布前文档检查，容易与代码脱节。

## 当前问题

- 部分页面布局与交互仍在持续优化（例如表格高度、按钮文案等体验项）。
- 大体量接口文档在 `06-开发参考` 中较长，检索成本较高。

## 待确认事项

- 是否需要引入细粒度权限模型（RBAC）。
- 是否需要将任务调度与 API 服务拆分为独立进程。
- 是否需要引入自动化回归（E2E）作为发版门禁。
