# 项目目标

## 1. 总体目标

建设一个“可运维、可扩展、可验证”的产业数据与 AI 应用平台，满足以下统一目标：

- 数据侧：稳定管理供应链主数据与采集结果
- 知识侧：沉淀结构化与非结构化知识并可检索
- AI 侧：支持 LangChain/RAG/智能体实用化
- 平台侧：具备模型、数据源、搜索、菜单、样式等运维能力

## 2. 分层目标

### 2.1 业务目标

- 支持 GYS + GAS 两套供应商体系的数据维护与查询。
- 支持爬取任务生命周期：创建、执行、监控、导入、重试。
- 支持供应商画像与产业图谱展示。

### 2.2 技术目标

- 采用 Node.js 24 + React + PostgreSQL（SeaboxSQL）稳定运行。
- 接口统一风格并接入鉴权中间件。
- 对外依赖（Auth/模型/MCP）异常时具备可读错误反馈。

### 2.3 AI 目标

- 支持多模型提供商配置与联通测试。
- 支持 LangChain 多轮对话和 RAG 对话。
- 支持精准寻源智能体（普通与流式输出接口）。

## 3. 验收目标（可验证）

1. 启动目标
- `npm run stable` 可在本地启动前后端。

2. 功能目标
- 主数据至少一条全链路 CRUD 成功。
- 爬取任务至少一条“创建->执行->日志/质量->导入”闭环。
- 知识库至少一条“接入->检索”闭环。
- LLM Wiki 至少一条“导入->条目->图谱”闭环。
- 精准寻源对话可返回结果或明确错误。

3. 文档目标
- 需求、架构、功能、测试文档必须映射到现有代码功能，不得停留模板级。
