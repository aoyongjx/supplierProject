# 业务流程

## 1. 核心端到端流程

1. 用户登录
- 前端获取登录地址，跳转 Auth，完成回调后进入系统。

2. 主数据准备
- 在供应链/供应商/OEM/画像模块维护基础数据，形成业务主表。

3. 采集任务执行
- 创建爬取任务并启动。
- 查看运行日志和质量报告。
- 对异常任务执行停止/重试。

4. 结果导入与核验
- 将任务输出导入业务表。
- 在列表页检索与抽样检查。

5. 知识增强
- 将文件、网页、文本接入知识库。
- 在 LLM Wiki 进行原始导入、条目沉淀与图谱同步。

6. 智能应用
- 在 LangChain/RAG/精准寻源中消费数据和知识，形成分析输出。

## 2. 关键分支流程

### 2.1 数据源调试分支

- 配置数据源连接
- 测试连接
- 浏览 schema 与表预览
- 执行受控 SQL 校验

### 2.2 模型能力配置分支

- 新增模型提供商
- 连通性测试
- 拉取模型清单
- 在对话/智能体中使用

### 2.3 系统可见性配置分支

- 通过菜单可见性配置控制导航入口展示
- 通过 UI 样式配置调整系统视觉主题

## 3. 异常流程

- 登录失败：回登录页并显示失败提示
- 任务失败：状态失败 + 日志可查 + 可重试
- 导入失败：返回原因并保留源任务上下文
- 模型不可用：对话返回错误并支持切换模型
- 外部服务异常：接口明确报错，不静默吞掉
